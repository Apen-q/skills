# daily-report-emailer

按下面流程工作，默认使用中文回复。

## 核心流程

1. 判断用户要做的是以下哪一类：
   - 生成日报，再发送
   - 已有内容，直接发送
   - 已有内容，先优化再发送
2. 提取或补齐最少必要信息：
   - 收件邮箱
   - 邮件主题
   - 邮件正文
   - 是否先优化
3. 如果用户只说“帮我生成今天工作总结”，先生成一版可发送的日报草稿，不立即发送。
4. 如果用户要求优化内容，先输出优化版，并明确标记：
   - 原始版
   - 优化版
   - 建议发送版本
5. 若信息已完整且用户已明确要求发送，直接发送；只有收件邮箱、正文或发送版本不明确时才追问。
6. 发送邮件时，优先运行 `scripts/send-report-email.ps1`。

## 对话规则

- 当信息缺失时，只问一个最关键的问题。
- 当用户已经给出明确收件邮箱和正文时，不重复追问无关信息。
- 当用户说“发送到 xxx 邮箱”时，优先使用当前会话里最近一份明确确认的正文。
- 当同时存在“原始版”和“优化版”时，若用户只说“发送”，先追问发送哪一版。
- 当用户明确说“不优化直接发送”时，不再改写正文，只允许补一个简短且专业的邮件主题。
- 当收件邮箱、正文和发送版本都已明确，且用户已表达发送意图时，直接调用发送脚本。

## 日报生成要求

- 默认结构：
  - 今日完成
  - 进行中事项
  - 风险或阻塞
  - 明日计划
- 如果用户只给出零散事项，整理成适合邮件发送的正式版本。
- 如果用户没有指定风格，默认输出简洁、专业、偏职场汇报风格。

## 内容优化要求

- 保留事实，不编造结果、数据、负责人或完成状态。
- 优先优化以下方面：
  - 结构清晰
  - 语气专业
  - 重点前置
  - 去除口语化和重复表达
- 如果原文非常短，例如“周五下午5:30来xxx开会”，可扩写成适合邮件发送的通知正文，但不得改变核心时间、地点、事项。

## 邮件发送要求

- 发送前整理出：
  - 收件人
  - 主题
  - 正文
  - 是否优化
- 优先发送纯文本邮件，除非用户明确要求 HTML。
- 使用 `scripts/send-report-email.ps1` 发送。
- 先从环境变量读取 SMTP 配置；变量说明见 `references/smtp-config.md`。
- 如果缺少 SMTP 配置，不假装已发送，明确指出缺少哪些配置。

## 推荐主题模板

- 日报：`今日日报 - {姓名或团队} - {YYYY-MM-DD}`
- 周报：`本周工作总结 - {姓名或团队} - {YYYY-MM-DD}`
- 通知：`通知：{事件摘要}`
- 会议：`会议通知：{日期时间} {事项}`

## 资源使用

- 发送邮件前，先阅读 `references/smtp-config.md` 了解所需环境变量和脚本调用方式。
- 需要实际发信时，运行 `scripts/send-report-email.ps1`。
- 若需要定时提醒，可运行 `scripts/send-daily-zimo-reminder.ps1` 或由宿主环境自行创建计划任务。

## 分享与安装

- 允许将整个 skill 文件夹分享给其他人安装。
- 分享时，不要把个人邮箱账号、SMTP 授权码或其他密钥写进脚本、`SKILL.md`、参考文档或资源文件。
- 其他使用者安装后，必须用他们自己的 SMTP 配置完成发信。
- 若对方需要长期使用，优先让对方运行 `scripts/set-smtp-env.ps1` 写入自己的环境变量。
- 若对方只需要临时发送一次，可直接在运行 `scripts/send-report-email.ps1` 时显式传入 SMTP 参数。

## 安全要求

- 不在 skill 文件中硬编码真实邮箱密码、客户端授权码或 API 密钥。
- 当用户在对话中提供真实授权码时，优先用于当前操作，不要把它回写进 skill 文件。
- 若发现 skill 中存在明文凭据，移除后再继续分享或安装。

## 示例

### 示例 1：先生成日报，再发送

用户：帮我生成今天工作总结。

处理：
- 先生成日报草稿。
- 如果用户继续说“帮我发送到 xxx 邮箱”，则补齐主题并进入发送准备。
- 如果用户继续说“将优化后的发送”，则先给优化版，再发送优化版。

### 示例 2：指定内容直接发送

用户：帮我发送“周五下午5:30来xxx开会”的指定内容到 xxx 邮箱。

处理：
- 将原句整理成一封简短正式邮件。
- 给出建议主题。
- 若用户未要求优化，则按原意直接发送。

### 示例 3：指定不优化

用户：把这段内容发到 xxx 邮箱，不进行优化直接发送。

处理：
- 不改正文内容。
- 仅补齐必要主题。
- 直接调用发送脚本。
